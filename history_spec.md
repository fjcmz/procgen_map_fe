# History Package Functional Specification

This document specifies the behavior implemented by classes under `es.fjcmz.lib.procgen.history` and its subpackages.

## 0. Complete class/member inventory (public vs private)

This section is intentionally explicit about visibility because most model classes expose state as public fields.

### 0.1 `es.fjcmz.lib.procgen.history`
- `HistoryRoot`
  - Public static fields: `INSTANCE`, `WORLD_REF`, `TIMELINE_REF`.
  - Public methods: `references()`.
  - Private fields/methods: none.
- `IdUtil`
  - Public static methods: `id(...)`, `idCustom(...)`.
  - Private static field: `joiner`.
- `HistoryGenerator`
  - Public static fields: `GSON`.
  - Public static methods: `main(...)`.
  - Private static fields: output path/file constants.
  - Private static methods: artifact emitters, graph emitters, connected-group helpers, histogram builders, statistics helpers.
  - Private static nested classes: `ConnectedCities`, `Histogram`.

### 0.2 `es.fjcmz.lib.procgen.history.physical`
- `World`, `Continent`, `Region`, `Resource`, `City`:
  - Public static `DEF` generable definitions.
  - Public fields used as serialized model (plus public transient runtime fields for non-persisted links/indexes).
  - Public `getId()/getReference()/setReference()` implementations.
  - `City` also exposes public behavior `canTradeMore()`.
- `WorldGenerator`, `ContinentGenerator`, `RegionGenerator`, `ResourceGenerator`, `CityGenerator`:
  - Public generator API methods: `getGeneratedType()`, `generate(...)`.
  - `RegionGenerator` has private helper methods: `assignNeighbours(...)`, `updatePotentialNeighbours(...)`.
- `CityVisitor`, `RegionVisitor`:
  - Public singleton instance fields (`cityVisitor`, `regionVisitor`).
  - Public traversal/selection APIs.
  - Protected helper methods for collection-based traversal/selection.

### 0.3 `es.fjcmz.lib.procgen.history.timeline`
- Model entities (`Timeline`, `Year`, `Foundation`, `Contact`, `Country`, `Illustrate`, `Wonder`, `Religion`, `Trade`, `Cataclysm`, `War`, `Tech`, `Conquer`, `Empire`):
  - Public static `DEF` generable definitions.
  - Public persistent fields + public transient runtime fields.
  - Public `getId()/getReference()/setReference()` as applicable.
  - Public nested enums for probabilistic domains.
- Generators (`TimelineGenerator`, `YearGenerator`, `FoundationGenerator`, `ContactGenerator`, `CountryGenerator`, `IllustrateGenerator`, `WonderGenerator`, `ReligionGenerator`, `TradeGenerator`, `CataclysmGenerator`, `WarGenerator`, `TechGenerator`, `ConquerGenerator`, `EmpireGenerator`):
  - Public generator API methods.
  - Additional private/protected selection/effect helpers per generator.
- `Merge`:
  - Public empty class; no fields or methods.

## 1. Generation root and top-level orchestration

## 1.1 `HistoryRoot`
- Defines the root graph with exactly two root references:
  - `world` (`World`, cardinality = 1)
  - `timeline` (`Timeline`, cardinality = 1)
- `references()` always returns `[WORLD_REF, TIMELINE_REF]` in that order.

## 1.2 `HistoryGenerator` (entrypoint + observability)
- Registers all history generables and all corresponding generators.
- Seeds `Randomizer` from `System.currentTimeMillis()` (commented option exists for deterministic seed `1337`).
- Generates one full `Root` with physical world and timeline.
- Emits output artifacts (JSON, CSV, Graphviz) including:
  - flattened generated references
  - per-entity dumps
  - timing statistics
  - histograms for population, counts of entities, tech levels, resources over time
  - graph of regional adjacency
  - graph of city contacts
  - connected components of city contact graph

## 1.3 `IdUtil`
- `id(Object... parts)`: joins by `_`.
- `idCustom(separator, parts...)`: joins by custom separator.
- Returns `null` if parts are null or empty.

## 2. Physical model (`history.physical`)

## 2.1 `World`
### Persistent fields
- `String id`
- `List<Continent> continents`
- `int endedOn`
- `String endedBy`

### Transient runtime indexes/state
- Back reference: `Root history`
- Geography indexes: `mapContinents`, `mapRegions`
- Civilization indexes: `mapCountries`, `mapCities`, `mapUsableCities`, `mapUncontactedCities`
- Cultural/event indexes: `mapIllustrates`, `mapUsableIllustrates`, `mapWonders`, `mapUsableWonders`, `mapReligions`, `mapWars`, `mapAliveWars`

### Generation
- Generated by `WorldGenerator`:
  - Sets `history`
  - `id = id("world", uuid)`
- Contains `continents` generated as random-size reference (`rndSize(8,5)`).

### Evolution
- `endedOn`/`endedBy` are set when cataclysm process removes all usable cities.
- Index maps are continuously mutated by yearly event generators and updater logic.

## 2.2 `Continent`
### Persistent fields
- `String id`
- `List<Region> regions`

### Transient
- `World world`

### Generation
- `ContinentGenerator` creates id `continent_<uuid>`, sets `world`, and inserts into `world.mapContinents`.
- `regions` generated via reference (`rndSize(100,1)`).

## 2.3 `Region`
### Persistent fields
- `String id`
- `Biome biome`
- `List<Resource> resources`
- `List<City> cities`
- `Set<String> neighbours` (IDs)

### Transient
- `Continent continent`
- `int neighboursCount` target adjacency count
- `boolean isCountry`
- `Country country`
- `boolean hasResources`
- `List<Region> neighbourRegions` direct references
- `List<List<Region>> potentialNeighbours` BFS layers by graph distance

### Generation
- `RegionGenerator` sets:
  - `id = region_<uuid>`
  - `biome` sampled from weighted `Biome`
  - `neighboursCount` sampled from roll `roll(2,4)-1`
  - adjacency assignment to up to `min(2, neighboursCount)` existing regions that still have adjacency capacity
- Adds to `world.mapRegions`.
- Recomputes `potentialNeighbours` for **all** regions after each region creation.
- References generated:
  - `resources` (`rndSize(10,1)`)
  - `cities` (`rndSize(20,1)`)

### Biome behavior
- Weighted biome probabilities with population growth multiplier:
  - `temperate(10, 1.5)`
  - `arid(10, 1.0)`
  - `desert(1, 0.3)`
  - `swamp(2, 0.5)`
  - `tropical(3, 0.7)`
  - `tundra(1, 0.3)`

### Evolution
- `isCountry` and `country` set when country founded.
- `hasResources` recomputed yearly (`resource.available >= TRADE_MIN` for any resource).

## 2.4 `Resource`
### Persistent fields
- `String id`
- `Type type`
- `int original`
- `int available`

### Generation
- `ResourceGenerator`:
  - `type` sampled by weighted enum
  - `id = resource_<type>_<uuid>`
  - `original = roll(10,10)+20`
  - `available = original`

### Resource taxonomy (weighted)
- Strategic/mineral/fuel: `copper(40)`, `iron(30)`, `aluminium(20)`, `uranium(3)`, `oil(20)`, `gas(30)`, `coal(40)`
- Agricultural: `cattle(20)`, `wheat(30)`, `rice(30)`, `sheep(20)`, `fruit(15)`
- Luxury: `silver(30)`, `gold(10)`, `diamonds(3)`, `silk(20)`, `incense(15)`

### Evolution
- Trade consumes `TRADE_USE=5` from `available` on both exchanged resources.
- War-ending of trade restores consumed amount.

## 2.5 `City`
### Persistent fields
- Core identity/state:
  - `String id`
  - `boolean founded`
  - `boolean contacted`
  - `int foundedOn`
  - `int destroyedOn`
  - `String detroyCause` (typo preserved from code)
- Demography:
  - `Size size`
  - `long initialPopulation`
  - `long currentPopulation`
- Relationship/event IDs:
  - `List<String> contacts`
  - `List<String> trades`
  - `List<String> illustrates`
  - `List<String> wonders`
  - `Map<String, Float> religions` (religionId -> local adherence fraction)
  - `List<String> cataclysms`

### Transient
- `Region region`
- `Foundation foundation`
- `Set<City> contactCities`
- `Set<Trade> tradeCities`
- `Map<Tech.Field, Tech> knownTechs`

### Generation
- `CityGenerator` sets:
  - `region`
  - `id = city_<uuid>`
  - `founded = false`
  - weighted `size`
  - `initialPopulation = size.initialPop(...)`
  - `currentPopulation = initialPopulation`
  - inserts into `world.mapCities`

### Size model
For each size: `(probabilityWeight, initialPopRoll, maxPopRoll, perYearRoll)`:
- `small(100, roll(2,10)+90, roll(10,1000)+20000, roll(1,20)+1)`
- `medium(40, roll(5,10)+200, roll(100,1000)+350000, roll(15,20)+10)`
- `large(15, roll(40,10)+400, roll(600,1000)+700000, roll(45,20)+20)`
- `metropolis(5, roll(100,10)+1000, roll(2000,1000)+3000000, roll(160,20)+30)`
- `megalopolis(1, roll(1000,10)+5000, roll(4000,1000)+8000000, roll(400,20)+50)`

### Trade capacity behavior
- Base max concurrent trades by size:
  - small 10, medium 15, large 20, metropolis 30, megalopolis 50
- For each tech in `TRADE_TECHS = {exploration, growth, industry, government}`:
  - capacity multiplier `*= (1 + tech.level/10)` if known
- `canTradeMore()` iff `trades.size() < round(adjustedCapacity)`

### Evolution
- Founded by `FoundationGenerator`: moved to usable/uncontacted maps.
- Contact status set via `ContactGenerator`.
- Population increases yearly by biome and growth-tech multipliers.
- Population decreased/destroyed by cataclysms.
- Religious composition propagates and strengthens yearly.
- Trade list and active trade references change via `TradeGenerator`/`WarGenerator`.

## 2.6 Visitors/utilities
- `CityVisitor`:
  - iterate all or usable cities
  - random selection with predicate, sampling without replacement over candidate list
- `RegionVisitor`:
  - iterate all regions
  - select up to N matching regions (randomized)
  - select one among up to N matches

## 3. Timeline model (`history.timeline`)

## 3.1 `Timeline`
### Persistent fields
- `int startOfTime`
- `List<Year> years`

### Transient
- `Root history`

### Generation
- `TimelineGenerator` sets `history` and picks `startOfTime = random.nextInt(2000) - 3000`.
- Generates fixed 5000 `Year` nodes via constant size reference.

## 3.2 `Year`
### Persistent fields
- `int year`
- `long worldPopulation`
- Per-year generated event/entity lists:
  - `foundations`, `contacts`, `countries`, `illustrates`, `wonders`, `religions`, `trades`, `cataclysms`, `wars`, `techs`, `conquers`, `empires`

### Transient
- `Timeline timeline`

### Generation order and size functions (as defined in references)
- `foundations`: random in `[0, max(2, toFound/300)-1]` where `toFound = totalCities - usableCities`
- `contacts`: `rndSize(30,2)`
- `countries`: `rndSize(10,0)`
- `illustrates`: random in `[0, max(2, usableCities/500)-1]`
- `wonders`: random in `[0, max(2, usableCities/500)-1]`
- `religions`: often zero (if two consecutive booleans true), else up to `max(2, withoutReligion/1000)` scaled by random double
- `trades`: `TRADES_ROLL = roll(6,10)`
- `cataclysms`: `rndSize(6,-3)`
- `wars`: random in `[0, max(2, countries/50)-1]`
- `techs`: `rndSize(5,1)`
- `conquers`: `rndSize(4,1)`
- `empires`: exactly `conquers.size()` when conquers exists, else 0

### Year pre-processing (`YearGenerator.generate` before sub-entity generation)
1. Abort generation if world already ended (`endedBy != null`).
2. Compute `year = timeline.startOfTime + timeline.years.size()`.
3. Set `worldPopulation` as sum of current populations of usable cities.
4. Increase populations.
5. Kill or retire illustrates (natural or war-related).
6. Propagate religions and recompute religion members.
7. End wars whose duration elapsed; refresh countries' `atWar` flags.
8. Recompute `region.hasResources` for trade viability.

## 4. Timeline entities and generators

## 4.1 `Foundation` / `FoundationGenerator`
### Fields
- transient: `Year year`
- persistent: `String id`, `String founded` (city id)

### Generation/effects
- If all cities already founded (`mapCities == mapUsableCities`), no foundation.
- Else choose random unfounded city.
- Create `foundation_<year>_<uuid>`.
- Mark city founded, stamp year, set back-reference.
- Add city to `world.mapUsableCities` and `world.mapUncontactedCities`.

## 4.2 `Contact` / `ContactGenerator`
### Fields
- transient: `Year year`
- persistent: `String id`, `String contactFrom`, `String contactTo`

### Selection logic
1. Prefer uncontacted city in multi-founded-city region.
2. Else uncontacted city in single-city region.
3. Else any usable city.
4. Target city chosen from reachable regions by BFS depth:
   - base depth 1
   - if source city has `exploration` tech, depth = `level + 1`
   - first founded, non-self, not-yet-contacted city found

### Effects
- Mark both cities contacted.
- Add symmetric links in both `contacts` ID lists and transient `contactCities` sets.
- Remove both from `mapUncontactedCities`.

## 4.3 `Country` / `CountryGenerator`
### Fields
- transient: `Year year`, `Region region`, `List<War> warCountries`, `Empire memberOf`, `Map<Tech.Field,Tech> knownTechs`
- persistent: `String id`, `Spirit spirit`, `String governingRegion`, `int foundedOn`, `boolean atWar`, `List<String> wars`, `List<String> empires`

### Spirit weights
- military(3), religious(3), industrious(3), neutral(9)

### Generation logic
- Select up to 5 random candidate regions where:
  - not already a country
  - all region cities are founded and contacted
- Pick one randomly among those.
- Create `country_<year>_<uuid>` and bind region ownership.
- Merge all city tech maps by max-level-per-field and assign unified map to:
  - country.knownTechs
  - each city in region (shared reference)
- Insert into `world.mapCountries`.

## 4.4 `Illustrate` / `IllustrateGenerator`
### Fields
- transient: `Year year`, `City originCity`
- persistent: `String id`, `Type type`, `String city`, `int yearsActive`, `String greatDeed`, `Integer diedOn`, `String deathCause`

### Type model
- religion(2, yearsActive roll(5,10))
- science(3, roll(5,6))
- philosophy(2, roll(5,12))
- industry(5, roll(5,6))
- military(5, roll(5,8))
- art(3, roll(5,10))

### Generation constraints/effects
- Eligible city: founded by current year and size in {large, metropolis, megalopolis}.
- Creates `illustrate_<year>_<type>_<uuid>`.
- Sets active duration and city bindings.
- Adds to city and world illustrate indexes (including usable map).

### Lifecycle
- `YearGenerator.killIllustrates`:
  - natural death when `birthYear + yearsActive <= currentYear`
  - possible war death (15% chance while active war affects origin country), limited by random `toKill` budget
- `CataclysmGenerator` may kill one reachable usable illustrate (50% chance to attempt).
- Used/consumed by Religion/Tech generation: removed from `mapUsableIllustrates` once producing a great deed.

## 4.5 `Religion` / `ReligionGenerator`
### Fields
- transient: `Year year`
- persistent: `String id`, `String founder`, `int foundedOn`, `String foundingCity`, `int members`

### Generation paths
1. **Found new religion** when there is usable `Illustrate` of type `religion` whose origin city has no religions:
   - create `religion_<year>_<uuid>`
   - founder = illustrate id; foundingCity from illustrate
   - set illustrate `greatDeed` and consume illustrate from usable map
   - initialize founding city's adherence with random [0.10, 0.49]
   - add religion to world registry
2. **Expand existing religion** otherwise:
   - pick random city with at least one religion
   - pick one of its religions
   - pick target city without that religion in same or neighbouring region
   - seed target adherence with random [0.01, 0.09]

### Ongoing dynamics (`YearGenerator.propagateReligions`)
- Single-religion cities drift +0.05 toward dominance until 0.9.
- If total adherence < 0.9, random existing religion gains +0.05.
- Religion member counts recomputed each year as sum of `cityPopulation * adherenceFraction` over usable cities.

## 4.6 `Trade` / `TradeGenerator`
### Fields
- transient: `Year year`, `City tradeCity1`, `City tradeCity2`, `Resource material1`, `Resource material2`
- persistent: `String id`, `int started`, `Integer ended`, `String endCause`, `String city1`, `String city2`, `Resource.Type resource1`, `Resource.Type resource2`

### Constants
- `TRADE_MIN = 10` minimal available resource to be tradable
- `TRADE_USE = 5` consumed resource amount per trade endpoint

### Generation logic/effects
- Choose source city from usable cities where:
  - `canTradeMore()`
  - source region currently `hasResources`
- Choose contacted target city in different region where:
  - target region `hasResources`
  - target `canTradeMore()`
- Choose one random source resource and one target resource each with `available > TRADE_MIN`.
- Create trade id `trade_<year>_<resource1>_<resource2>_<uuid>`.
- Decrease each chosen resource available by 5.
- Add trade IDs and transient trade references to both cities.

### Termination
- War can end selected cross-region trades and restore resource availability (+5 each side).

## 4.7 `Cataclysm` / `CataclysmGenerator`
### Fields
- transient: `Year year`
- persistent: `String id`, `Type type`, `Strength strength`, `String city`, `double killRatio`, `long killed`

### Type model
Each type has `(probability, killRatioRoll, canDestroyWonder)`:
- earthquake(50, roll(10,3), true)
- volcano(30, roll(7,5), true)
- tornado(50, roll(8,4), true)
- asteroid(1, roll(20,2), true)
- tsunami(5, roll(15,2), true)
- flood(70, roll(6,3), false)
- heat_wave(80, roll(3,4), false)
- cold_wave(80, roll(3,4), false)
- drought(70, roll(5,4), false)

### Strength weights
- local(100), regional(10), continental(2), global(1)

### Generation flow
- Pick random usable city as epicenter.
- Sample type/strength.
- `killRatio = roll/100`.
- Create id `cataclysm_<year>_<type>_<strength>_<uuid>`.
- Apply casualties by affected scale:
  - global => global + continental + regional + local cascading due fall-through switch
  - continental => continental + regional + local
  - regional => regional + local
  - local => local only
- Casualty per city = `round(currentPopulation * killRatio)`.
- If casualties >= population: city destroyed (`currentPopulation=0`, destroyed stamp, removed from usable map).
- Optionally kill one reachable usable illustrate.
- If type can destroy, may destroy one reachable non-destroyed wonder.
- If no usable cities remain, set world end (`endedOn`, `endedBy`).

## 4.8 `War` / `WarGenerator`
### Fields
- transient: `Year year`
- persistent: `String id`, `Reason reason`, `int started`, `String aggressor`, `String defender`, `int lasts`

### Reason weights
- expansion(20), religion(10), resources(20), vengance(2)

### Generation flow
- Choose aggressor from countries not currently at war.
- Choose defender among neighbouring regions that are countries and not in same empire as aggressor.
- Create id `war_<year>_<reason>_<lasts>_<uuid>`, where `lasts = roll(2,5)`.
- Mark both countries at war; register in both countries and world maps (`mapWars`, `mapAliveWars`).
- Attempt to terminate some inter-country trades between both regions:
  - probability threshold per candidate trade: random percent in [8,17]
  - ended trade gets `ended=war.started`, `endCause=war.id`
  - resource availability restored

### Ongoing/ending
- `YearGenerator.endWars` removes alive wars once `started + lasts < year` and clears atWar.
- Active wars reassert `atWar=true` each year for involved countries.

## 4.9 `Tech` / `TechGenerator`
### Fields
- transient: `Year year`
- persistent: `String id`, `Field field`, `int level`, `String discoverer`

### Field weights
- science(3), military(3), industry(3), energy(2), growth(2), exploration(1), biology(1), art(1), government(1)

### Utility methods in `Tech`
- `mergeAllTechs(Collection<Map<Field,Tech>>)`:
  - union by field, keeping max-level tech per field
- `getNewTechs(originalTechs, newTechs)`:
  - delta map where field absent or level increased

### Generation logic/effects
- Pick random usable illustrate; if none -> no tech.
- Determine eligible fields from `illustratesToTech` multimap derived from static mapping.
- Discoverer = illustrate id.
- Determine known tech map:
  - city map by default
  - country map if city belongs to country
  - empire founder country map if country in empire
- New tech level = existing level in field + 1.
- Create id `tech_<year>_<field>_<level>_<uuid>`.
- Insert/replace in known-tech map.
- Set illustrate `greatDeed`; consume illustrate from usable map.

## 4.10 `Conquer` / `ConquerGenerator`
### Fields
- transient: `Year year`, `War inWar`, `Country conquerorCountry`, `Country conqueredCountry`
- persistent: `String id`, `String war`, `String conqueror`, `String conquered`, `Map<String,Object> acquired`

### Generation logic
- Picks one finishing war where `war.started + war.lasts == currentYear`.
- Randomly selects winner between aggressor/defender.
- Creates id `conquer_<year>_<uuid>`.
- Removes finished war from alive-war map.

### Effects
- Both countries `atWar = false`.
- Tech assimilation:
  - merge conqueror and conquered tech maps (max-level each field)
  - compute acquired delta
  - store acquired tech IDs under `acquired["techs"]`
- Empire implications:
  - if conquered belonged to empire: remove it; if empire drops to one member, dissolve empire and stamp destruction by conqueror
  - if conqueror belongs to empire: add conquered into that empire (countries/reach/members, plus memberOf pointer)

## 4.11 `Empire` / `EmpireGenerator`
### Fields
- transient: `Year year`, `Country founder`, `Set<Country> members`
- persistent: `String id`, `int foundedOn`, `String foundedBy`, `Integer destroyedOn`, `String conqueredBy`, `Set<String> countries`, `Set<String> reach`

### Generation logic
- Triggered from a conquer in current year where conqueror is not already in empire.
- Create id `empire_<year>_<uuid>`.
- Founder = conqueror country.
- Initialize with conqueror + conquered as members/countries/reach.
- Add empire id to both countries' `empires` lists.
- Set both countries `memberOf` to new empire.

## 4.12 `Merge`
- Placeholder empty class; no functionality currently implemented.

## 5. Cross-entity lifecycle narrative

1. **Physical scaffold creation**: world -> continents -> regions -> resources/cities; region graph and BFS neighbourhood layers are formed during generation.
2. **Timeline start**: random start year in approximately [-3000, -1001], then up to 5000 year slots.
3. **Per-year preprocessing** updates world state before creating that year’s events (population, religion propagation, war cleanup, resource flags).
4. **City awakening** through foundations gradually moves cities into active simulation.
5. **Network formation** through contacts; exploration tech expands contact radius over region-neighbour levels.
6. **Political formation** with countries when whole regions become founded/contacted.
7. **Culture & knowledge loop**:
   - illustrates appear in larger established cities;
   - illustrates are consumed to produce religions or technologies;
   - tech affects growth, trade capacity, and contact reach.
8. **Economy loop**:
   - trades consume resources while active;
   - wars can cancel trades and return resources.
9. **Conflict loop**:
   - wars begin among neighbouring countries;
   - finishing wars can generate conquests;
   - conquests can found/expand/destroy empires.
10. **Disaster loop**:
    - cataclysms reduce populations, may destroy cities/wonders, and can kill illustrates.
11. **Termination**:
    - once no usable cities remain, world gets `endedBy` and `endedOn`, and subsequent year generation stops.

## 6. Known implementation-specific quirks (spec-level notes)

- Wonder city predicate precedence means **all large/metropolis cities qualify regardless of resource richness**, while `megalopolis` additionally requires `>1` high-original resources due to `&&` binding tighter than `||`.
- Cataclysm kill switch intentionally/implicitly uses switch fall-through to apply broader-than-selected scopes.
- `City.detroyCause` field name is misspelled in the model and should be treated as canonical serialized key.
- `City.Size.maxPop(...)` exists but is not currently enforced in yearly growth.
- `Year.worldPopulation` is measured before annual population increase in that year-generation cycle.
